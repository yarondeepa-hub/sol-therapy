# LinkedIn Post Workflow

> תהליך עבודה משותף בין הכותב (Copywriter) לשומר הסף (Gatekeeper) ליצירת פוסט לינקדאין.

---

## סקירת התהליך

```
[אתם] נותנים כיוון ונושא
          ↓
[Copywriter] כותב טיוטה ראשונה
          ↓
[Gatekeeper] בודק ומאשר/מחזיר להערות
          ↓
[Copywriter] מתקן (אם צריך)
          ↓
[Gatekeeper] אישור סופי
          ↓
[אתם] מפרסמים
```

---

## שלב 1: הכנה

### לפני שמתחילים

שני הסוכנים חייבים לקרוא:
- `C-core/project-brief.md` - מי אתם ומה אתם עושים
- `C-core/voice-dna.md` - איך המותג שלכם מדבר
- `C-core/icp-profile.md` - מי קהל היעד שלכם
- `M-memory/learning-log.md` - מה למדנו מפעמים קודמות

### הגדרת הפרויקט

צרו תיקייה חדשה ב-`O-output/` עם שם ברור:
```
O-output/
└── 01-linkedin-post-[topic]/
    ├── copywriter-draft.md    ← גרסת הכותב
    ├── gatekeeper-review.md   ← גרסת שומר הסף
    └── final-post.md          ← גרסה סופית לפרסום
```

3. **כותב טיוטה ראשונה** - לפי העקרונות שלמד

## שלב 3: שומר הסף בודק

### מה שומר הסף עושה

1. **קריאה ראשונה** - רושם ראשוני תוך 5 שניות

3. **החלטה** - מאושר / לתיקון / לשיחה

## שלב 4: לולאת שיפור אוטומטית (Automatic Revision Loop)

> **חשוב:** הלולאה הזו קורית אוטומטית ברקע — בלי התערבות אנושית!

### איך זה עובד

```
[Copywriter] כותב טיוטה v1
         ↓
[Gatekeeper] בודק
         ↓
   ┌─────────────┐
   │ APPROVED?   │
   └─────────────┘
     │         │
    YES        NO
     │         │
     ↓         ↓
  סיום    [Gatekeeper] שולח פידבק ספציפי
                  ↓
           [Copywriter] מתקן אוטומטית → v2
                  ↓
           [Gatekeeper] בודק שוב
                  ↓
              (חוזר עד אישור או 3 ניסיונות)
```

### הכללים

1. **אוטומטי לגמרי** — הכותב והשומר עובדים ביחד בלי לחכות לאינפוט אנושי
2. **מקסימום 3 סיבובים** — אם אחרי 3 גרסאות עדיין לא אושר → ESCALATE לאדם
3. **היסטוריית גרסאות** — כל גרסה נשמרת (v1, v2, v3)
4. **רק הגרסה הסופית מוצגת** — האדם רואה רק את התוצאה המאושרת

### מה הכותב עושה בלולאה

```
אם קיבלתי REVISIONS NEEDED:
1. קרא את הפידבק הספציפי
2. תקן רק את מה שנדרש (אל תשנה מה שעובד)
3. שמור כגרסה חדשה (v2, v3...)
4. החזר לשומר הסף מיד
```

### מה שומר הסף עושה בלולאה

```
אם שלחתי REVISIONS NEEDED:
1. צרף פידבק ספציפי ומיידי
2. העבר ישירות לכותב (לא לאדם!)
3. המתן לגרסה חדשה ובדוק שוב

אם זה סיבוב 3 ועדיין לא מאושר:
→ ESCALATE לאדם עם סיכום הבעיה
```

### פורמט היסטוריית גרסאות

```markdown
## Version History

| גרסה | סטטוס | הערות |
|------|-------|-------|
| v1 | REVISIONS | Hook חלש, חסרים מספרים |
| v2 | REVISIONS | Hook טוב יותר, CTA לא ברור |
| v3 | APPROVED | ✅ מוכן לפרסום |
```

```markdown
**תאריך אישור:** [תאריך]

---

---

*נבדק על ידי: Gatekeeper Agent*
```
